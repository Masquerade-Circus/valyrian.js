# 8.2. Money and Number Formatting (`valyrian.js/money`)

This module keeps money values precise with cents-based arithmetic, locale-aware formatting, and practical parsing helpers for common input patterns.

## `Money` Value Object

```ts
import { Money } from "valyrian.js/money";

const subtotal = Money.fromDecimal(19.99);
const tax = Money.fromDecimal(1.6);
const total = subtotal.add(tax);
```

Core APIs:

* `Money.fromCents(cents)`
* `Money.fromDecimal(value, decimalPlaces?)`
* `toCents()`
* `toDecimal(decimalPlaces?)`
* `add`, `subtract`, `multiply`, `divide`

`divide(0)` throws.

## `NumberFormatter`

`NumberFormatter` is lower-level numeric cleanup + formatting utility.

```ts
import { NumberFormatter } from "valyrian.js/money";

const formatted = NumberFormatter.create("$1,234.56").format(2, { currency: "USD" }, "en-US");
```

Useful APIs:

* `set(value, shiftDecimal?)`
* `format(digits?, options?, locale?)`
* `fromDecimalPlaces(decimalPlaces)`
* `toDecimalPlaces(decimalPlaces)`
* `shiftDecimalPlaces()`

## Parsing and Formatting Helpers

```ts
import { formatMoney, parseMoneyInput } from "valyrian.js/money";

const parsed = parseMoneyInput("$1,234.56", { locale: "en-US", decimalPlaces: 2 });
const view = formatMoney(parsed, { currency: "USD", locale: "en-US", digits: 2 });

const euros = parseMoneyInput("1.234,56 €", { locale: "de-DE" });
const euroView = formatMoney(euros, { currency: "EUR", locale: "de-DE", digits: 2 });
```

Parsing notes:

* uses `Intl.NumberFormat(locale).formatToParts(...)` to detect decimal and group separators when a locale is provided
* strips spaces, NBSP-style separators, and currency symbols
* includes fallback handling for common `.` and `,` separator patterns, including a lone comma used as a decimal separator
* supports common inputs such as `$1,234.56`, `1.234,56 €`, and `$1,234.56` with `es-MX`, but does not attempt to cover every financial notation or accounting format
* invalid input returns `Money.fromCents(0)`

Formatting notes:

* `Money` keeps cents as the default internal scale of 2 decimal places
* `digits` controls display precision and does not mutate or rescale the stored cents value
* `decimalPlaces` controls how a `Money` value is converted from minor units before formatting; it defaults to `2` for compatibility

## `v-money` Directive

`v-money` keeps an input in formatted currency while storing minor units in your model. Use `decimalPlaces` to match the currency scale, for example `0` for JPY or `2` for USD.

```tsx
<input
  v-money={{
    model: form.state,
    field: "amountInCents",
    currency: "USD",
    locale: "en-US",
    decimalPlaces: 2
  }}
/>
```

`v-money` preserves any existing `oninput` handler and calls it after updating and reformatting the input value.
