# 7.1.1. Node Runtime APIs (`valyrian.js/node`)

`valyrian.js/node` is the Node-facing server runtime surface of Valyrian.

It prepares the runtime environment that SSR and request-scoped server execution need, then exposes the rendering, DOM, and storage APIs that keep browser/server behavior aligned under one runtime model.

## Runtime Setup Side Effects

Importing `valyrian.js/node` initializes the Node runtime surface:

* `global.document` (lightweight DOM adapter)
* `global.FormData`
* `global.sessionStorage` and `global.localStorage` backed by `ServerStorage`

Import it before using server-side modules that expect these globals.

## `render(...nodes)`

`render` is the server render entry point for the shared runtime model. It mounts content into a server DOM container, serializes HTML, and unmounts internally.

```tsx
import { render } from "valyrian.js/node";

const html = render(<main><h1>Hello SSR</h1></main>);
```

You can pass multiple nodes; they are rendered in order.

Reader note: because `render` unmounts internally, each call is isolated and does not reuse DOM state from previous calls. That per-call isolation is what makes request-by-request SSR safe.

## DOM Transform Utilities

Exports:

* `htmlToDom`
* `domToHtml`
* `htmlToHyperscript`
* `domToHyperscript`

```ts
import { htmlToDom, domToHtml } from "valyrian.js/node";

const dom = htmlToDom("<section><p>Hello</p></section>");
const html = domToHtml(dom);
```

## Build Helpers Also Exported Here

`valyrian.js/node` also exports build helpers from the same package surface:

* `inline`
* `icons`
* `sw`

Use them from tooling scripts (see [./7.3-pwa-and-build-tooling.md](./7.3-pwa-and-build-tooling.md)).

## Request-Scoped Storage

`ServerStorage` is part of the server runtime surface for per-request storage isolation.

Use `ServerStorage.run(() => { ... })` per incoming request to avoid storage leakage between concurrent SSR requests.

See [./7.2-isomorphic-networking-and-storage.md](./7.2-isomorphic-networking-and-storage.md) for practical request-scoped usage.
